## 摘要

摊小二是面向小微商户的 SaaS 商城，本文档定义一套基于「**双积分 + 四类奖励 + 五星等级**」的营销体系，用于推动复购、激励分享、构建分层用户运营。

核心机制：

| 模块 | 一句话说明 |
|---|---|
| 双积分 | 消费积分（仅消费抵现）+ 推广积分（可提现 / 可转消费积分） |
| 直推奖励 | 推荐 1 个下级买，按商品「推 N 反 1」配置返推广积分 |
| 队列分配 | 自然用户购买（无上级）→ 该笔奖励按队列规则返给"在排队的人" |
| 团队极差 | 沿推荐链向上发，按星级递减分润 |
| 星级积分池 | 商户额外提取一部分订单金额 → 池子定期分给星级用户 |

## 一、积分体系

| 类型 | 来源 | 用途 | 限制 |
|---|---|---|---|
| **消费积分** | 商品级配「每元返多少消费积分」 | 仅在该商户消费抵现 | ❌ 不可提现 / ❌ 不可跨商户 |
| **推广积分** | 直推 / 队列返 / 团队极差 / 星级积分池 | ① 转换为消费积分（默认 1:1，商户可改）<br>② 在该商户处线下提现（满商户配置门槛可申请） | 跨商户隔离，每商户独立账户 |

## 四、订单成交时奖励处理顺序

依次结算（基于**实付价**）：

1. **消费积分入账**（买家本人，按商品配置）
2. **直推 / 队列分配**（按第五节规则二选一执行）
3. **团队极差奖**（仅当买家有上级链时触发）
4. **入星级积分池**（若商品在白名单 → 实付 × 入池比例 进池）

## 五、直推 + 插队 + 自然推三机制（核心）

> **作用范围**：仅作用于「参与推 N 反 1 的商品」。
> **队列粒度**：每商户每商品**独立一条队列**。

### 5.1 三种触发场景

| 买家身份 | 处理路径 |
|---|---|
| **有上级**（被推荐过来） | 走「**直推**」：奖励直接按 N 反 1 配置返给上级；奖励同时算上级 1 次累计；上级若在 B 层 → 立即升 A 层 |
| **自然用户**（无上级） | 走「**自然推**」：奖励返给当前商品队列的队首（A 层优先，A 空给 B 层）；同时该买家进入该队列尾部 B 层 |
| **B 层老用户自购同款** | 走「**插队**」：自购奖按本人当前累计位置返给本人；本人 B → A 层 |

### 5.2 队列分两层（同一商品队列内）

- **A 层**：做过主动行为（自购 / 推下级成交）的用户 → 优先返奖
- **B 层**：仅自然消费过 1 单进队 → A 空了才轮到
- A 层内部按"晋升 A 层时间"先后排序

### 5.3 累计 + 出队规则

- 每次返奖按该用户当前累计次数定**位置**和**比例**：第 1 次 → 商品 N 反 1 第 1 个比例，第 2 次 → 第 2 个，依此类推
- 累计**满 N 次**（N = 商品配的 N）→ **出队**，累计共获 100% 商品价的推广积分
- B 层老队头**永不过期**：等不到下一笔自然用户购买就一直挂着，商户/商品冷清不补不发

### 5.4 完整跑例（商品 N=3，比例 30/30/40）

| 事件 | A 层 | B 层 | 谁累计 +1 | 备注 |
|---|---|---|---|---|
| 1 自然买 | [] | [1] | — | 1 进 B 层尾，无上家可返 |
| 2 自然买 | [] | [1,2] | 1（位 1=30%）| 自然推：返给 B 队首 1 |
| 3 自然买 | [] | [1,2,3] | 1（位 2=30%）| 同上 |
| 4 自然买 | [] | [2,3,4] | 1 满 → **出队**，得 100 积分 | 1 累计 3 次（30+30+40） |
| 5/6/7 依次 | [] | [4,5,6,7] | 2 满 → 出 → 3 接队 | 同样 3 步出队 |
| 7 推 8 买（直推）| [7] | [3,4,5,6] | 7（位 1=30%）| 直推奖给 7，7 升 A 层 |
| 7 推 9 买 | [7] | [3,4,5,6] | 7（位 2=30%）| 直推奖给 7（仍 A 队首）|
| 第 10 自然用户买 | [] | [3,4,5,6,10] | 7 满 → **出队** | A 优先；10 进 B 尾 |

## 六、团队极差奖（递减模式）

每笔订单成交触发一次（不论买家自然 / 被推荐），**沿推荐链向上**：

```
设链路 1 → 2 → 3 → 4 → 5 → 6 → 7 → 8（8 是买家或推荐链末端）

step 1: 8 拿直推奖（已在第五节处理，跳过）
step 2: 8 是星级用户吗？
  - 否 → 跳到 step 3
  - 是 N 星 → 8 拿 N% 实付价为推广积分
            记录"链路下方已发星级 = N"
step 3: 沿链向上找最近的「比已发星级更高」的星级用户
  - 找到 M 星 → 拿 (M% - 已发%) = 差额
  - 更新"已发 = M"，继续 step 3
  - 找不到 → 结束
```

**算法要点**：每个上级只拿"自己星级% **减去** 链路下方所有已发出星级%"，互不重复。

**示例**：5 星之上若再有 5 星 → 上面的 5 星拿 (5%-5%) = **0%**（被吃掉）。这鼓励上级保持高于下级。

## 七、星级评定（每商户独立）

- **AND 条件**：直推下级数 ≥ N **AND** 团队链路销售份数 ≥ M
- **每个星级各一组 N、M**，商户后台配
- **终生制**：达到后**不会**因不活跃降级
- **团队链路销售份数** = 自己 + 所有下级（不限层）累计销售的「参与推 N 反 1」商品份数
- **跨商户隔离**：用户在 A 商户的星级 ≠ B 商户的星级

### 升星门槛建议默认值（仅参考，商户可改）

| 星级 | 直推下级 | 团队链路销售 |
|---|---|---|
| 1 星 | ≥ 2 | ≥ 3 份 |
| 2 星 | ≥ 3 | ≥ 9 份 |
| 3 星 | ≥ 5 | ≥ 27 份 |
| 4 星 | ≥ 8 | ≥ 81 份 |
| 5 星 | ≥ 12 | ≥ 243 份 |

## 八、星级积分池

### 8.1 商户配置

| 字段 | 说明 |
|---|---|
| 参与商品白名单 | 多选（开了"参与积分池"开关的商品才进白名单） |
| 入池比例 | 实付价 × ____%（建议 5%） |
| 可参与瓜分的星级 | 多选（如仅 3-5 星，或 1-5 星全部）|
| 池子分配方式 | A) **全员均分**：池子 ÷ 所有参与用户人数<br>B) **按星级均分**：池子 ÷ 参与星级数 = 每星级桶；桶内再均分给该星级用户 |
| 结算频率 | 每周 / 每两周 / 每月 / 自定义 cron |
| 抽奖中奖占比 | 每次结算抽 ____% 用户中奖（建议 5%） |

### 8.2 每次结算前商户选

| 模式 | 说明 |
|---|---|
| **均分** | 按"分配方式"直接发放给所有参与用户 |
| **抽奖** | 按占比抽中若干人，中奖者按"分配方式"瓜分池子 |

### 8.3 结算后

- 池清零，下个周期重新累积
- 每订单字段 `已纳入星级积分池` 标记，防重复计入
