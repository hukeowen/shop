# 摊小二 营销引擎 — 商户使用手册（v6）

> 本手册写给开通营销功能的商户。配套需求文档见 `design/marketing-system-v6.md`，
> 工程进度见 `design/promo-engine-progress.md`。

---

## 1. 你能用这套系统做什么

把"卖货 → 拉人 → 留客 → 复购"四件事用一套规则串起来：

| 模块 | 一句话 |
|---|---|
| 双积分 | 顾客买东西既得**消费积分**（仅在你店里抵现），又能通过分享赚**推广积分**（可线下提现） |
| 直推奖 / 推 N 反 1 | 顾客拉来 1 个新人买你商品，按你配置的比例返推广积分给推荐人 |
| 队列分配 | 如果买家没有上家（自然进店），奖励留给排队中的"老顾客" |
| 团队极差 | 沿推荐链向上分润，星级越高拿得越多（递减算法，不重复） |
| 星级积分池 | 你额外抽商品 N% 进池子，定期分给高星级用户（按月 / 抽奖均可） |

---

## 2. 入口怎么进

商户登录商户 H5（`/m/`）后：

```
首页 → 我的 → 营销配置（v6）         ← 商户级总配置
首页 → 我的 → 提现审批               ← 顾客提现申请审批
商品列表 → 编辑商品 → 营销配置（v6） ← 商品级开关
```

## 3. 第一次配置走完一遍（5 分钟）

### 3.1 进 "营销配置（v6）"

页面分四块：

**① 平台星级**
- **星级数量**：填 1–10（建议 5）。每升一级，团队极差比例越高
- 然后给每个星级配 3 个数：
  - **极差抽成 %**：本星级用户从其链下消费里能拿的总比例（建议 1 / 2 / 3 / 4 / 5）
  - **直推人数**：升到这一星需要直推几个有效下级
  - **团队销售额**：升到这一星需要整条推荐链累计卖多少元

> 默认 5 / [1,2,3,4,5]% / 升星门槛 (2,3) (3,9) (5,27) (8,81) (12,243) — 点开就是这套，
> 不想改直接保存即可。

**② 推广积分**
- **推广 → 消费 转换比例**：用户主动转换时按这个换算（默认 1:1）
- **提现门槛（元）**：低于此金额不允许申请（默认 100 元）

**③ 星级积分池**（默认关）
- **开启**：勾上才走入池
- **入池比例 %**：参与商品的实付价 × 此比例进池（建议 5%）
- **可参与瓜分的星级**：勾出哪些星级有资格领（多选；建议至少勾 3 星以上）
- **分配方式**：
  - **全员均分**：池子 ÷ 中奖人数 = 每人金额
  - **按星级均分**：池子先按星级数等分到桶；桶内再均分给该星级中奖人（高星单人多拿）
- **结算 cron**：默认每月 1 号 0 点（`0 0 0 1 * ?`）
- **抽奖中奖占比 %**：LOTTERY 模式下随机抽多少比例的人中奖（建议 5%）
- **cron 自动结算模式**：默认 FULL 全员均分；选 LOTTERY 则按上面占比随机抽

**④ 积分池运营**（开启后才显示）
- **当前余额** + **最近结算**：实时数字
- **全员均分结算 / 抽奖结算**：随时手动触发，不等 cron
- **结算历史**：最近 10 轮，含模式 / 参与人 / 中奖人 / 总金额

底部 "保存配置" → 全量上行后端，立即生效。

### 3.2 商品级开关：编辑商品 → 营销配置（v6）

每个商品独立配，不受总开关影响：

| 字段 | 含义 |
|---|---|
| 消费积分倍率 | 每元返多少消费积分（0=关闭返积分） |
| 参与推 N 反 1 | 是否走"推 N 反 1"队列分润 |
| N 值 | 推几个才返完。例 4 = 推 4 单返完 100% 商品价 |
| N 个返佣比例 | 长度 = N 的数组，例 25/25/25/25 等额；前低后高也行 |
| 参与星级积分池 | 该商品销售是否进池 |

**典型配置**：
- 普通商品：消费积分倍率 1，其它全关 → 顾客拿消费积分，无推广分润
- 推广爆款：N=4, 比例 [30, 30, 20, 20]，参与池 → 直推走队列、销售进池
- 团购品：消费积分倍率 0.5，仅参与池 → 不发直推、纯靠池分

## 4. 一笔订单成交后会发生什么

```
订单标记 PAID（trade 模块）
  │
  ▼
MerchantPromoOrderHandler.afterPayOrder
  │
  ├─ ① 消费积分入账（按商品 ratio）
  ├─ ② 直推 / 队列 / 自然推（按商品 tuijianEnabled）
  │     - 买家有上家 → 上家拿 N 反 1 第几次的奖
  │     - 买家自然 / 自购同款 → 队首拿奖 / 自己升 A 层拿奖
  ├─ ③ 入池累计（按商品 + 商户 poolEnabled）
  ├─ ④ 星级回算（参与商品份数 +1，沿链向上）
  └─ ⑤ 团队极差递减（按商户 commissionRates，订单总额一次）
```

每步都用 `(userId, sourceType, orderId)` 三元键判重；订单重放 / 多 SKU 同 SPU 不会重复发奖。

## 5. 提现审批怎么走

进 "我的 → 提现审批"，4 个 tab：

| Tab | 含义 | 你能做的操作 |
|---|---|---|
| 待审 | 顾客刚提交的申请，积分已从他余额扣减 | 通过 / 驳回 |
| 待打款 | 已通过；等你线下转账 | 标记已打款 |
| 已完成 | 已转账 | 仅查看 |
| 已驳回 | 已退还积分 | 仅查看 |

每条卡片显示：金额 / 用户 ID / 申请时间 / 处理时间 / 备注。

> **驳回 = 自动退还** 积分到顾客推广积分余额，无需手动还原。

## 6. 用户那一面

顾客进入用户 H5 后：

| 页面 | 能干啥 |
|---|---|
| 首页底部 "我的" | 看到 "我的钱包（双积分）" 入口 |
| 我的钱包 | 看推广 / 消费 余额 + 流水 + 推广→消费 转换 |
| 申请提现 | 输入金额 → 校验门槛 / 余额 → 提交（积分立刻扣减） |
| 我的钱包 → 操作 → 申请提现 / 积分池历史 | 同上 |

**邀请关系怎么建**：分享链接带 `?inviter=USER_ID` 或 `?referrerUserId=USER_ID`，
被邀请人进店后系统自动暂存；他登录后第一次进 "我的首页" 就会自动绑定上级（仅生效一次，
终生制不可改）。

## 7. 积分池怎么自动结算

系统已经预置了一个定时任务（`promoPoolSettleJob`），默认**关闭**：

```
后台 → 基础设施 → 定时任务 → 找到 "积分池自动结算 Job" → 状态改为 NORMAL
```

启用后每小时扫一次，对每个商户判断：
- 该商户启用了池子？
- 上次结算之后，cron 表达式（`pool_settle_cron`）的下一次触发点 ≤ 当前时间？

满足两条就按商户配置的 `pool_settle_mode`（FULL / LOTTERY）自动跑一次结算。

> 不启用也没关系：营销配置页里 "全员均分结算 / 抽奖结算" 按钮可以手动触发。

## 8. 常见疑问

**Q：用户买了商品但我没看到推广积分发出去？**
A：检查这三件：① 商品的 "参与推 N 反 1" 开了吗 ② 买家有没有绑定上级（看 "推荐链 → 上级"）
③ 上级是否已退队（累计满 N 次会标记 EXITED 不再返奖）。

**Q：极差递减拿不到钱？**
A：极差只发给"星级 > 链下方已发星级"的用户。如果链上每个人都同星，只有最下面那位拿到，
上面全部 0% — 这是 v6 文档的明确规则（鼓励上级保持高于下级）。

**Q：积分池抽奖人数 0？**
A：参与人数 × 抽奖中奖占比 % 向下取整 = 0 时，自动跳过该轮。提高占比或等参与人变多。

**Q：用户驳回提现后是否还能再申请？**
A：可以。驳回后他的活跃申请清空，余额已退还，可立即再次申请（要满足门槛）。

**Q：为什么我配置的 cron 表达式没生效？**
A：检查 ① 是 Quartz cron（6 段，含秒）不是 Linux cron（5 段）② "积分池自动结算 Job"
是否启用为 NORMAL ③ 上次结算时间 + cron 下一次触发点是否真的早于当前时间。
